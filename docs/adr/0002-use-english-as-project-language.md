# 0002. Use English as the project language

Date: 2026-07-08

## Status

Accepted

## Context

The package is intended for public open-source development in the `sebastian-software` GitHub organization and for publication to npm. Contributors and users should be able to read documentation, issues, releases, and code comments without needing German context.

## Decision

The project language is English.

Repository documentation, code comments, commit messages, release notes, npm package metadata, GitHub issues, pull requests, and ADRs should be written in English unless there is a specific external reason to do otherwise.

## Consequences

Public project artifacts are accessible to the broader Node.js and open-source ecosystem.

Internal planning conversations can still happen in another language, but repository-facing artifacts should be English.
