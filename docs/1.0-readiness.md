# 1.0 Readiness

This document tracks the evidence needed before declaring `1.0.0`.

The public preview release remains a prerequisite. Published npm install checks, published tarball inspection, and the final `1.0.0` Release Please pull request cannot be completed until the npm package records and Trusted Publishing setup are bootstrapped in [#9](https://github.com/sebastian-software/pdfium-node/issues/9).

## Stable Surface

These surfaces are the intended `1.0.0` stability promise after preview validation:

- package name: `@sebastian-software/pdfium-node`;
- supported platforms: Linux x64 glibc and macOS arm64;
- minimum Node.js version: `>=22`;
- public exports: `renderPdfThumbnails`, `ErrorCodes`, and `PdfiumNodeError`;
- public API: `renderPdfThumbnails(pdf, options)`;
- public options: `pages`, `maxWidth`, `scale`, `format`, `quality`, `timeoutMs`, `maxPixels`, `maxInputBytes`, and `background`;
- public result type: page number, dimensions, MIME type, and image bytes;
- typed error codes exported through `ErrorCodes`;
- error metadata property: `metadata`;
- task-level API shape, with no raw PDFium handles or PDFium pointer access.

The public export list and exact error code strings are covered by `test/api.test.mjs`. Type declarations for the same surface are shipped from `packages/pdfium-node/src/index.d.ts`.

## Remaining 1.0 Evidence

The following evidence is still required before `1.0.0`:

- public preview packages published through the npm workflow;
- npm install smoke tests from the published package on Linux x64 glibc and macOS arm64;
- README quick start verified against the published package;
- published package tarball inspection for wrapper files, native binaries, PDFium metadata, licenses, and third-party notices;
- Release Please pull request for `1.0.0`;
- npm Trusted Publishing run for `1.0.0`.
